# 优化资源库搜索功能计划

## 需求分析
用户要求：
1. 在设置页面增加"完美匹配"/"大致匹配"选项
2. 不符合搜索词的结果不显示
3. 全面优化搜索算法

## 当前状态
- `core/search_engine.py`：独立搜索引擎模块，已实现多维度匹配评分
- `core/settings_manager.py`：设置管理器，单例模式，统一管理所有设置
- `gui/pages/setting_page.py`：设置页面，5个标签页（基础/下载/界面/高级/隐私）
- 当前搜索逻辑：通过分数过滤（score > 0），冲突检测使用扣分机制

## 实施方案

### 1. 新增搜索模式设置项

#### 1.1 更新 `core/settings_manager.py`
- 新增默认设置：
  - `search_mode`: 搜索模式（"完美匹配"/"大致匹配"，默认"大致匹配"）
  - `search_suggestions_enabled`: 搜索建议开关（默认 True）
  - `search_max_results`: 搜索结果最大数量（默认 100，0 表示不限制）
  - `search_smart_repair`: 智能关键词修复开关（默认 True）
- 添加到热缓存列表（高频读取）

#### 1.2 更新 `gui/pages/setting_page.py`
- 在"高级"标签页新增"🔍 搜索设置"分组（放在日志设置下方）
- 添加搜索模式下拉框（"完美匹配"/"大致匹配"）
- 添加搜索建议开关
- 添加搜索结果数量限制（SpinBox，0=不限制）
- 添加智能关键词修复开关
- 每个设置项附带说明文字
- 连接自动保存信号

#### 1.3 更新 `gui/pages/setting_page.py` 的 `_collect_settings` 和 `_refresh_ui`
- 收集搜索相关设置
- 刷新时恢复搜索设置控件状态

### 2. 优化搜索引擎算法

#### 2.1 修改 `core/search_engine.py` 的 `search()` 方法
- 从设置读取 `search_mode` 参数
- 根据模式调用不同的过滤逻辑

#### 2.2 新增 `_filter_strict()` 方法（完美匹配模式）
- **学科冲突**：如果搜索了明确学科，资源学科不匹配则直接过滤（不返回）
- **年级冲突**：如果搜索了明确年级，资源年级不匹配则直接过滤
- **版本冲突**：如果搜索了明确版本，资源版本不匹配则直接过滤
- **关键词覆盖**：至少 50% 的搜索关键词需要匹配到资源
- **最低分数**：分数 >= 200 才返回

#### 2.3 优化 `_search_and_score()` 方法（大致匹配模式）
- 保持现有逻辑，但提高最低分数阈值到 50
- 保留冲突扣分机制（而非直接过滤）

#### 2.4 优化 `_calculate_score()` 方法
- 提高学科匹配权重：+100 → +150
- 提高年级匹配权重：+80 → +100
- 提高版本匹配权重：+100 → +150
- 多关键词匹配加成：+30 → +50
- 学科不匹配惩罚：-500 → -800（完美模式直接过滤）
- 年级冲突惩罚：-300 → -500
- 版本冲突惩罚：-300 → -500

#### 2.5 新增关键词匹配度检测
- 计算匹配的关键词数量 / 总关键词数量
- 完美模式要求匹配度 >= 50%
- 大致模式要求匹配度 >= 25%

### 3. 全面优化搜索体验

#### 3.1 搜索建议增强
- 在 `resource_page.py` 中集成搜索建议下拉框
- 输入时实时显示建议列表
- 支持键盘上下键选择和回车确认

#### 3.2 搜索空结果优化
- 当搜索结果为空时，显示友好提示
- 根据搜索模式给出建议：
  - 完美模式：建议切换到"大致匹配"或修改关键词
  - 大致模式：建议修改关键词或检查拼写

#### 3.3 搜索结果摘要优化
- 显示"找到 X 个完美匹配的资源，Y 个近似匹配的资源"
- 在完美模式下，如果无结果但存在近似结果，提示切换模式

## 假设与决策
1. 搜索模式设置放在"高级"标签页的日志设置下方，作为独立分组
2. 完美匹配的最低分数阈值为 200，大致匹配为 50
3. 完美模式下，学科/年级/版本冲突直接过滤（而非扣分）
4. 完美模式要求关键词匹配度 >= 50%
5. 搜索结果摘要功能留待后续实现（本次先完成核心搜索算法）
