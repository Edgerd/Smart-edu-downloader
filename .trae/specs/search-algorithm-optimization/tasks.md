  - [ ] 1.1 修复 _repair_search_term：使用正则边界匹配，避免"数学"被拆为"数"+"学"导致残留"学"
    - **实现细节**：在提取学科时，先遍历完整学科名列表（['数学', '语文', '英语', '道德与法治', '信息技术', '体育与健康']），使用 `re.search(r'学科名', term)` 进行匹配；如果匹配成功则移除该完整词；之后才处理缩写映射；使用 `re.sub(r'(?<![一二三四五六七八九])数(?![学])', '', term)` 类似的正则边界匹配，确保"数学"中的"数"不被误识别
  - [ ] 1.2 修复 _extract_keywords：先匹配完整学科名再匹配缩写，册次扩展只保留完整形式
    - **实现细节**：关键词提取时，先检查完整词（如"上册"、"下册"、"数学"），如果完整词已存在，不再添加其缩写形式（如"上"、"数"）；移除 `_extract_keywords` 中 `keywords.append(vol)` 简写形式的代码；移除 `keywords.extend(['下', '下册'])` 中的简写部分
  - [ ] 1.3 新增 _optimize_keywords：去除部分词（"学"被"数学"包含则移除）、去除过短词
    - **实现细节**：实现方法 `_optimize_keywords(keywords: List[str]) -> List[str]`：
      1. 先按长度降序排序关键词
      2. 遍历每个关键词，如果关键词 A 包含关键词 B（A != B），则移除 B
      3. 移除长度 < 2 的关键词（但保留数字和"上"/"下"等特殊情况）
      4. 去重后返回
  - [ ] 1.4 search() 方法在提取关键词后调用 _optimize_keywords
    - **实现细节**：在 `search()` 方法中，`keywords = self._extract_keywords(repaired_keyword)` 之后添加 `keywords = self._optimize_keywords(keywords)`

- [ ] Task 3: 搜索缓存机制
  - [ ] 3.1 实现 SearchCache 类（LRU，最多 20 条，过期 5 分钟）
    - **实现细节**：在 search_engine.py 中新增 `SearchCache` 类：
      - 内部使用 `OrderedDict` 存储缓存项
      - 每个缓存项包含：`{'keyword': str, 'results': List[Dict], 'timestamp': float, 'hit_count': int}`
      - `get(keyword) -> Optional[List[Dict]]` 方法：检查是否存在、是否过期（5分钟），未过期则更新 hit_count 并移到末尾（LRU），返回结果
      - `put(keyword, results)` 方法：如果超过 20 条，移除最久未使用的（OrderedDict 头部）
      - `clear()` 方法：清空所有缓存
  - [ ] 3.2 search() 方法优先检查缓存
    - **实现细节**：在 `search()` 方法开头，先调用 `self._cache.get(keyword)`，如果命中则直接返回；未命中则正常执行搜索，搜索完成后调用 `self._cache.put(keyword, results)`
