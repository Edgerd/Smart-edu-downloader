# 智能检索框增强 Spec

## Why
当前主页智能检索框仅支持 URL 链接下载，无法处理中文关键词搜索。用户希望输入课本名称后能像资源库一样搜索，并支持在搜索结果中上下切换选择目标课本后下载。

## 交互流程

```
用户输入课本信息或链接
        ↓
点击"智能检索并下载"
        ↓
   ┌────────────────────────────────┐
   │ 自动检测输入类型                │
   │                                │
   │ URL 链接 → 直接下载（现有流程）  │
   │ 中文关键词 → 资源库搜索          │
   └────────────────────────────────┘
        ↓（搜索完成）
按钮变为三按钮：
  "就是这个！"  "上一个"  "下一个"
封面预览区 → 显示第1个结果
处理结果区 → 显示第1个课本名
        ↓
用户用 ↑↓ 键或按钮切换选择
        ↓
点击"就是这个！" → 下载当前选中的课本
        ↓
下载完成后 → 按钮恢复为"智能检索并下载"
```

## What Changes
- **智能输入检测**：自动区分 URL 链接和中文关键词，关键词走资源库搜索流程
- **按钮动态变换**：点击"智能检索并下载"后，搜索完成时按钮自动拆分为"就是这个！" + "上一个" + "下一个"
- **搜索结果导航**：支持键盘上下键和按钮切换搜索结果
- **封面预览联动**：切换搜索结果时封面预览区实时更新
- **处理结果联动**：切换搜索结果时处理结果区显示当前课本名

## Impact
- Affected specs: None (new feature)
- Affected code: `gui/pages/home_page.py`（主要修改，`_create_button_area`、`_smart_search_and_download`、新增导航方法），`resources/images/titles/上一个_svg.svg`、`下一个_svg.svg`（已存在）

## ADDED Requirements

### Requirement: 智能输入检测与路由
用户点击"智能检索并下载"后，系统 SHALL 自动检测输入内容类型并路由。

#### Scenario: 检测到 URL 链接
- **WHEN** 用户输入内容以 `http://`、`https://` 或 `www.` 开头
- **THEN** 系统走现有 URL 下载流程（行为不变，按钮不拆分）

#### Scenario: 检测到中文关键词
- **WHEN** 用户输入内容不以 `http://`、`https://`、`www.` 开头
- **THEN** 系统调用资源库 `search_resources` 进行关键词搜索
- **AND** 搜索完成后进入导航模式

### Requirement: 搜索完成后按钮变换
搜索完成后，系统 SHALL 将"智能检索并下载"按钮替换为三按钮组合。

#### Scenario: 按钮变换
- **WHEN** 资源库搜索返回结果列表
- **THEN** 原"智能检索并下载"按钮隐藏
- **AND** 显示三按钮组合："就是这个！"（确认下载）、"上一个"（↑箭头图标）、"下一个"（↓箭头图标）
- **AND** 封面预览区显示第一个结果的封面
- **AND** 处理结果区显示第一个结果的课本名称
- **AND** 当前选中索引设为 0

#### Scenario: 搜索无结果
- **WHEN** 资源库搜索返回空结果
- **THEN** 按钮保持"智能检索并下载"不变
- **AND** 处理结果区显示"未找到相关课本"

### Requirement: 搜索结果导航
系统 SHALL 在搜索完成后提供搜索结果导航功能。

#### Scenario: 用户点击"下一个"按钮
- **WHEN** 用户点击"下一个"按钮（↓ 向下箭头图标）
- **THEN** 当前选中索引 +1，封面预览更新为下一个结果的封面
- **AND** 处理结果更新为下一个结果的课本名称
- **AND** 如果当前已是最后一个结果，则切换到第一个（支持循环）

#### Scenario: 用户点击"上一个"按钮
- **WHEN** 用户点击"上一个"按钮（↑ 向上箭头图标）
- **THEN** 当前选中索引 -1，封面预览更新为上一个结果的封面
- **AND** 处理结果更新为上一个结果的课本名称
- **AND** 如果当前已是第一个结果，则切换到最后一个（支持循环）

#### Scenario: 用户按键盘方向键
- **WHEN** 用户按下键盘上键（↑）
- **THEN** 行为等同于点击"上一个"按钮
- **WHEN** 用户按下键盘下键（↓）
- **THEN** 行为等同于点击"下一个"按钮
- **AND** 方向键仅在导航模式下生效

### Requirement: 确认下载
系统 SHALL 在用户确认选择后触发下载。

#### Scenario: 用户点击"就是这个！"
- **WHEN** 用户点击"就是这个！"按钮
- **THEN** 系统以当前选中搜索结果的资源 URL 触发下载流程
- **AND** 下载完成后三按钮组合隐藏，恢复"智能检索并下载"按钮

#### Scenario: 无选中结果
- **WHEN** 用户点击"就是这个！"但当前无选中结果
- **THEN** 系统显示提示"请先选择一个课本"

### Requirement: 按钮图标
系统 SHALL 使用 SVG 图标作为导航按钮。

#### Scenario: 渲染导航按钮
- **WHEN** 搜索完成后显示导航按钮
- **THEN** "上一个"按钮使用 `resources/images/titles/上一个_svg.svg` 渲染
- **AND** "下一个"按钮使用 `resources/images/titles/下一个_svg.svg` 渲染
- **AND** 图标使用主题强调色（`#2078DA`）渲染
